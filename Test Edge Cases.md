Do not only test the “normal” case.

For **comparisons**, test:
- greater than
- less than
- equal to

For **loops**, test:
- empty array
- one item
- many items

For **forms**, test:
- valid input
- blank input
- weird input like spaces or symbols
    

This is huge. A lot of student code works only for the “happy path,” meaning the one nice clean example. 

Real code has to survive chaos.
