Use [extra care] when using single quotation marks with possessives and contractions in strings, because **JavaScript interpreters always look for the first closing single or double quotation mark to match an opening single or double quotation mark**. 

For example, consider the following statement:
```js
document.write('<p>My mom's favorite color is blue.</p>');
//causes ERROR
```

---
**The Problem**: A JavaScript interpreter assumes that the literal string ends with the apostrophe following “mom” and looks for the closing parentheses for the document.write() statement immediately following “mom’”. 

To get around this problem, 

Include an [[escape character]] before the apostrophe in “mom’s”.  

Placing a backslash before an apostrophe tells JavaScript interpreters that the apostrophe is to be treated as a regular keyboard character, such as _a_, _b_, _1_, or _2_, and not as part of a single quotation mark pair that encloses a text string.

The backslash in the following statement **tells the JavaScript interpreter to print the apostrophe following the word “mom” as an apostrophe**.
```js
document.write('<p> My mom\'s favorite color is blue.</p>');
```
---

>NOTE: 
>If you place a backslash before any character other than those listed in [[String Escape Sequences (Table)]] , the **backslash is ignored**.

---

Notice that one of the characters generated by an escape sequence is the backslash. Because the escape character itself is a backslash, you must use the escape sequence `\\` to include a backslash as a character in a string. 

For example, to include the path:
`“C:\Users\me\Documents\Cengage\WebWarrior\JavaScript\”` in a **string**, you **must** include two backslashes for every single backslash you want to appear in the string, as in the following statement:
```js
document.write("document.write("<p>My Javascript files are located in
	C:\\Users\\Documents\\Cengage\\WebWarrior\\
	JavaScript\\</p>");
```
