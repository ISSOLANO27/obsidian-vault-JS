XHTML tried to clean up HTML by enforcing strict rules.

> They aimed for correctness while HTML aimed for continuity. They did not say "*do not make mistakes*" they said, "*Mistakes Will Happen, this is how we will handle it*...". This is why updates where slow in order to make sure legacy content would not break.

The problem was that it didn’t work with older websites.
Too much would have broken at once.

Developers rejected it, and the web moved on.
