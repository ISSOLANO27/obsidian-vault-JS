Aka *Multitier Client/Server System* or *n-tier/Server System*

consists of 3 pieces:
-  [[Client tier]] (UI on web browser)
   - client makes a request of a database on a web server

-  [[Processing tier or Middle tier]] (processing bridge)
   - handles interaction between the web browser client and the data storage tier.
   
-  [[Data storage tier]]
	- The *database tier* in a *two tier client/server* system is split into the **processing** and **data storage** tier.

![[Pasted image 20260211002228.png]]

Essentially, the client here makes a request of a database on a web server. The process interior performs any necessary processing or calculations based on the request from the client tier. Then information is your writer from Oregon writes information to the data storage tier.

> NOTE: 
> In a Client-Server Model, the architecture  is more of a physical arrangement where the client and the server are two separate computers. 
> Here, it is more conceptual than physical
