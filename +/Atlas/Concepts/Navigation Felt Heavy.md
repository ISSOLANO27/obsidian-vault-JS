Before JavaScript became a reliable, persistent client-side scripting tool:

- **Every interaction required a full page reload.** Clicking a link, submitting a form, or requesting new data meant the browser had to send a request to the server, wait for the server to process it, and then reload the entire page.
    
- **No dynamic updates.** The page couldn’t change parts of itself (like updating a shopping cart or showing new comments) without refreshing everything.
    
- **Slow and clunky user experience.** Even small actions felt “heavy” because they triggered the full request/response cycle, which was slower and visually disruptive.
