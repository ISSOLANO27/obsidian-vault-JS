The eTextbook _New Perspectives on HTML5 and CSS, Comprehensive_ introduces CSS by breaking it into sessions and tutorials—covering styles, colors, typography, pseudo-elements, selectors, validation, accessibility, and more. Within this framework, the mention of “over 50 CSS modules” highlights that:

- [[CSS is modularized]] 
    
- **Examples from the book’s structure**:
    
    - _Session 2.1_ covers styles and colors.
        
    - _Session 2.2_ focuses on typography and selectors.
        
    - _Session 2.3_ introduces pseudo-classes and pseudo-elements.
        
    - Appendices cover accessibility, validation, and design principles. These are practical teaching slices of the broader CSS modules defined by the W3C.
        
- **Why modularization matters**: It allows the CSS Working Group to update and evolve parts of CSS independently. For example, new features in _CSS Grid Layout_ or _Flexbox_ can be standardized without waiting for every other CSS feature to be finalized.
    

### Putting it together
So when the text says “over 50 CSS modules,” it’s emphasizing that CSS is not a single frozen document but a living collection of specifications. The textbook mirrors this modular approach by teaching CSS in chunks—colors, fonts, selectors, pseudo-elements, accessibility—each corresponding to one or more official CSS modules. This helps learners see CSS as a flexible, evolving toolkit rather than a rigid set of rules.
